Incohérences majeures identifiées

Clique sur les problèmes que tu veux prioriser

1

Headers incohérents

Detail & Quick Create ont un header avec gradient riche + avatar. Task Detail a un header simple plat. Le bouton ✕ est rond/blur vs carré/gris.

2

Confirmations destructrices = window.confirm()

Les suppressions/annulations utilisent le confirm natif du navigateur au lieu d'une modale stylée. Rupture visuelle complète.

3

Pas de feedback de chargement

Aucun spinner ni état disabled pendant les appels API. L'utilisateur peut double-cliquer sur "Enregistrer" sans retour visuel.

4

Duplication Tâche

Quick Create a un toggle RDV/Tâche qui crée un mini-formulaire tâche. Task Detail est une modale séparée. Deux chemins pour la même chose.

5

Accessibilité manquante

Pas de focus trap, pas d'attributs ARIA, pas de navigation clavier dans les modales. Le tab sort de la modale.

6

UX mobile à harmoniser

Pas de swipe-to-close, targets tactiles trop petites (28px), clavier virtuel qui cache les inputs.

7

Labels & structure de formulaire

Task Detail a des labels explicites (.m-field-label). Detail et Quick Create n'en ont pas — les sections utilisent des titres incohérents.

8

Toast trop fugaces

Les erreurs disparaissent en 3.5s. Un seul toast à la fois — les messages se perdent si plusieurs actions rapides.
